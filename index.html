EPIC Viewer

Custom, combined front-end for multiple back-end tools

Visit Site

Upon entering the Design Services department with Deluxe Corporation, I quickly identified several web-based tools that could be consolidated and improved upon with front end web development.

While the back-ends of all services were isolated from my department, I was able to reverse-engineer all the main API routes to the back-end services, and coded an improved version of a primary tool which was called "EPIC Viewer" (to this day, I still do not know why the original author called it that).

Using a combination of vanilla JavaScript, jQuery, and plugins such as Data Tables and a keyboard shortcut helper library, I was able to consolidate the several tools into one improved web app.

In addition to the improved web app, I was able to introduce several new tools to this app specifically designed to improve productivity of the entire team of 40+ individuals.

This project is source controlled by Git, and is viewable on my Bitbucket profile page.

CardGames

Front-end only card game score-keeper

Visit Site

This project started as my way of escaping responsibilities of being score keeper in my family's favorite card games. One such example is 3-13. This game as 11 pre-defined rounds, with scores being kept for each round, the winner being the person with the lowest score.

Here is some info from the readme:

This app will keep track of scores for several card games. At the moment, these games are supported by the app:

  • Three-Thirteen

Each of these games has its own page to be used to interact with the app. Each game has a pop-up which will display the rules of the game, and you can configure the number of players for each game, the names, and add scores for each round.

The app will take care of keeping a running score, keep track of which level each person is on, as well as determine a winner (or winners, if there is a tie).

I am working separately on a Node version, which will allow other people to view the scorecard, while the scorekeeper is left with the ability to enter the scores.

This project is listed under the TwilightCoders Github account due to my relationship with that organization started by my best friend, and mentor, Dale.
